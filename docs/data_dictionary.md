# Data Dictionary -- Fuzzy & Co. Ecommerce Funnel Analysis

This document describes the structure, fields and relationships of the datasets used in the ecommerce funnel analysis.

---

## orders

| Field | Description |
|------|-------------|
| order_id | Unique identifier for each order (Primary Key) |
| created_at | Timestamp when the order was placed |
| website_session_id | Identifier for the associated website session (Foreign Key) |
| user_id | Identifier for the user who placed the order (Foreign Key) |
| primary_product_id | Identifier for the primary product in the order (Foreign Key) |
| items_purchased | Number of items included in the order |
| price_usd | Total revenue generated from the order |
| cogs_usd | Cost of goods sold for the order |

---

## order_items

| Field | Description |
|------|-------------|
| order_item_id | Unique identifier for each order item (Primary Key) |
| created_at | Timestamp when the item was created |
| order_id | Identifier for the order this item belongs to (Foreign Key) |
| product_id | Identifier for the product (Foreign Key) |
| is_primary_item | Binary flag (1 = primary item in order, 0 = secondary item) |
| price_usd | Revenue generated by the item |
| cogs_usd | Cost of goods sold for the item |

---

## order_item_refunds

| Field | Description |
|------|-------------|
| order_item_refund_id | Unique identifier for each refund (Primary Key) |
| created_at | Timestamp when the refund was issued |
| order_item_id | Identifier for the refunded order item (Foreign Key) |
| order_id | Identifier for the order the refunded item belonged to (Foreign Key) |
| refund_amount_usd | Monetary value of the refund |

---

## products

| Field | Description |
|------|-------------|
| product_id | Unique identifier for the product (Primary Key) |
| created_at | Timestamp when the product was launched |
| product_name | Name of the product |

---

## website_sessions

| Field | Description |
|------|-------------|
| website_session_id | Unique identifier for each session (Primary Key) |
| created_at | Timestamp when the session started |
| user_id | Identifier for the user (Foreign Key) |
| is_repeat_session | Binary flag (1 = returning user, 0 = new user) |
| utm_source | Traffic source (e.g., Google, Facebook) |
| utm_campaign | Marketing campaign name |
| utm_content | Specific ad or content variant |
| device_type | Device category (mobile or desktop) |
| http_referer | Referring URL |

---

## website_pageviews

| Field | Description |
|------|-------------|
| website_pageview_id | Unique identifier for each pageview (Primary Key) |
| created_at | Timestamp when the pageview occurred |
| website_session_id | Identifier for the associated session (Foreign Key) |
| pageview_url | URL path of the page viewed |

---

## Analytical Views

Four views were built on top of the raw tables in `sql/00_views_setup.sql`.
Run this file before any analysis scripts.

| View | Description |
|---|---|
| session_funnel | One row per session showing binary flags for each funnel step reached: saw_product, saw_cart, saw_shipping, saw_billing, saw_thank_you. Also includes session_revenue. |
| product_page_visit_count | One row per session per product page visited, with funnel step flags scoped to that product page visit. |
| multivariant_test | One row per session showing the landing page and funnel step flags for landing page performance analysis. |
| user_attribution | One row per user showing first-touch and last-touch channel, campaign, ad content, session ID, and conversion status. Built using ROW_NUMBER() to guarantee one row per user regardless of timestamp ties. |
